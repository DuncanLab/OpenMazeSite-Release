Introduction

The advent of 3D virtual environments has given researchers the ability to study human cognition in novel ways, providing insight into many important scientific questions. However, too often these experimental designs are stifled by lack of resources or expertise.

OpenMaze is an open source software package that provides and easy an intuitive platform to quickly create 3D environment tasks without programming expertise.

Configuration File

The configuration file is the most important file in the OpenMaze codebase when it comes to designing your own experiments. Here you can change up almost everything about how OpenMaze works just by changing a few numbers or words! The following section demonstrates exactly how to accomplish this.

Blocks

Blocks allow for organization and clear visualization of trial data. The order of blocks can be changed within the configuration file, which allows for easy manipulation of trial order when designing an experiment.

Parameter Default Type Description
Block Order 1 int Defines the order of the blocks
Block List [1] Array Can be labeled to aid organization of trials, defines trial order for each block

Trials

This section of the configuration file allows for organization of information within trials. This includes parameters that create a 3D environment tailored to your task, selection among different goal types and the timing of your trials.

Parameter Default Type Description
Index 4 int
Header Tutorial4: Goals String Define what message you would like to have at the top left corner of the trial screen
Time allotted -1 int Here you can determine the length of each trial, in seconds or if you need to press space to complete the trial
Environment Type 1 int Select the environment type the trial should take place in
RandomLoc 0 int Here you can enter 1 if you would like to randomly generate your position in the environment
CharacterStartPos [-10,10] array Defines the position within the environment where you appear at the beginning of the trial
Initial angle 50 int Here you can choose the first view of the environment that is presented when the trial begins, rotated about the initial character position
Time to rotate 0.5 int Enter the duration of rotation, at the beginning of the trial your view will be rotated 360 degrees around the environment
Sides 4 int The number of sides that will be present in local environment
Radius 25 int The radius of the local environment
Wall height 2 int The height of the sides that were previously defined
Wall colour 000000 Hex (String) Define the colour of the sides using the HTML colour code
Ground tile sides 4 int The number of tile sides you would like for the ground, enter 0 if solid ground colour is preferred
Ground tile size 1 int Define the size of the previously mentioned ground tiles
Ground colour 000000 Hex (String) Define the colour of the ground using HTML colour code
Active goals 0 int Enter numbers of objects you would like to appear in the environment that will be visible and collectible
Invisible goals 0 int Enter numbers of objects you would like to appear in the environment that will be invisible and collectible
Inactive goals 0 int Enter numbers of objects you would like to appear in the environment that will be visible, however not collectible
Quota 0 int Here you can enter the total number of objects that need to be collected for the trial to be completed before the allotted time runs out
Landmarks 0 int Enter numbers of landmarks that you would like to appear in the environment

Goals

In this section you can define different aspects of your goal objects. From physical characteristics such as size and colour to others including location and sound. You can create a wide variety of goal objects of varying salience to suit your task.

Parameter Default Type Description
Tag 1 int Label your item for quick reference
Type Pickup String Define what operation you want pickup item to fulfill
Colour 000000 Hex (String) Define the colour of the object using HTML colour code
Location [0,0,0] Array Define the location of the object in the 3D environment
Length 1 int Length of object in x-direction
Width 1 int Length of object in y-direction
Height 1 int Length of object in z-direction
Initial rotation 0 int Here you can adjust the rotation of the object about its defined location
Sound location Eat String Enter in sound tag that you previously stored to reference the sound

Landmarks

This section allows you to define the characteristics of your landmarks. This includes parameters that regulate location, size and colour. With the ability to manipulate features of landmarks you can further define the environment to reach your desired goal.

Parameter Default Type Description
Location [5,5,0] Array Define the location of the landmark in the 3D environment
Length 1 int Length of the landmark in x-direction
Width 1 int Length of landmark in the y-direction
Height 1 int Length of the landmark in the z-direction
Type Pickup String Define what operation you want pickup item to fulfill
Colour 000000 Hex (String) Define the colour of the landmark using HTML colour code
Initial rotation 0 int Here you can adjust the rotation of the landmark about its defined location

Tasks

The configuration file is the most important file in the OpenMaze codebase when it comes to designing your own experiments. Here you can change up almost everything about how OpenMaze works just by changing a few numbers or words! The following section demonstrates exactly how to accomplish this.

Create Experiment

A straightforward method for creating your own experiment is to copy and paste the sample configuration file. Then change the values for the different sections including landmarks, goal objects and trials all of which are organized by blocks.

Add image trials

To add an image trial, just three lines are required. The first can be used to indicate trial number and to label for your convenience. The second line should include the file location of your desired image and lastly the third line determines for how long the image will be presented.

Parameter Default Type Description
File location Tutorial1/Welcome.png string Enter the location of the image that you would like to add by referring to the file and image name
Time allotted -1 int Here you can determine the length of each trial, in seconds or if you need to press space to complete the trial

Add Goal Objects

Goal objects can be added by simply copying and pasting the sample text in the goal data section. Make sure to add a comma after the original text, then change parameter values as desired. The object number can then be added to active, inactive and invisible goals in the required trial.

Parameter Default Type Description
Tag 1 int Label your item for quick reference
Type Pickup String Define what operation you want pickup item to fulfill
Colour 000000 Hex (String) Define the colour of the object using HTML colour code
Location [0,0,0] Array Define the location of the object in the 3D environment
Length 1 int Length of object in x-direction
Width 1 int Length of object in y-direction
Height 1 int Length of object in z-direction
Initial rotation 0 int Here you can adjust the rotation of the object about its defined location
Sound location Eat String Enter in sound tag that you previously stored to reference the sound

Add Landmarks

Landmarks can be added by simply copying and pasting the sample text in the landmark data section. Make sure to add a comma after the original text, then change parameter values as desired. The landmark number can then be added under the landmark parameter in the required trial.

Parameter Default Type Description
Location [-60,-60, 15] Array Define the location of the landmark in the 3D environment
Length 30 int Length of the landmark in x-direction
Width 45 int Length of landmark in the y-direction
Height 30 int Length of the landmark in the z-direction
Type Sphere String Here you can select a 3D shape to be used as the landmark
Colour ff0000 Hex (String) Define the colour of the landmark using HTML colour code
Initial rotation 50 int Here you can adjust the rotation of the landmark about its defined location
